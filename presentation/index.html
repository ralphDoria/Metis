The premise

Short-form feeds aren't entertainment. They're neural conditioning that target the same circuits as casinos and slot machines.

  • Reward (OFC, ACC) lights up on every novel hook
  • Salience (insula, STS) keeps the next swipe loaded
  • Control (dlPFC) gets quieter the longer you scroll
Result

You haven't decided to keep watching. You're just downstream of a system that has already decided for you.

What if the feed could see itself?

Predict the brain's response to each reel as you watch it — then surface that signal back, in time to matter.

The pipeline

   video bytes
        ▼
   TribeV2       fMRI prediction model on Modal A10G GPU
        ▼
   voxels (T × 20,484 cortical vertices)
        ▼
   parser        Destrieux atlas → 7 ROIs → 9 pattern templates
        ▼
   verdict       {score, label, patterns, feedback}

score = (reward + 0.5·salience) / (control + ε) · discretized to low / average / elevated / high.

The latency problem

Modal inference takes seconds, not milliseconds. A naïve "capture → wait → maybe skip" loop is too slow — the user has already finished the reel.

  • Cold start: 5–30 s
  • Warm call + network: 4–8 s
  • Average reel: 15–45 s
Insight

Predict on reel N + 1 while you watch reel N. By the time you scroll, the verdict is already cached.

Speculative prefetch

 viewport
 ┌─────────────────────────┐
 │                         │   ← prefetchIO band  (rootMargin 200%)
 │   ┌─────────────────┐   │
 │   │   ▶ reel N (active) ┼──→ verdict cached or arriving
 │   └─────────────────┘   │
 │   ┌─────────────────┐   │
 │   │   reel N+1      ┼──→ capture started speculatively
 │   └─────────────────┘   │
 │   ┌─────────────────┐   │
 │   │   reel N+2      ┼──→ queued
 │   └─────────────────┘   │
 └─────────────────────────┘

Two IntersectionObservers — activeIO at threshold 0.6 (foreground), prefetchIO at rootMargin: 200% (incoming). Capture starts the moment a reel enters the prefetch band.

First 10 seconds only

The hook lives in the opening seconds. After 10 s the user is committed and the verdict can't drive a useful intervention.

  • Prefetched (paused) reels: seek to 0, capture the full first 10 s
  • Active reels: capture the remainder of the first-10s window
  • Past 10 s: skip processing entirely — save Modal compute
  • Boundary detected via requestVideoFrameCallback — recorder stops the moment currentTime ≥ 10

Cancel on user skip

If you scroll past a reel before our verdict lands, we kill the GPU run. 


@app.post("/process")
async def process(video: UploadFile, job_id: str | None = None):
    fc = Tribe().infer.spawn(video_bytes)
    if job_id:
        ACTIVE_JOBS[job_id] = fc
    result = await asyncio.to_thread(fc.get)   # ← cancellable
    ...

@app.post("/process/cancel/{job_id}")
def cancel_process(job_id: str):
    fc = ACTIVE_JOBS.pop(job_id, None)
    if fc: fc.cancel()                          # ← Modal kills the GPU job

Content script emits cancel_capture on 'lost'; SW aborts the fetch and fires POST /process/cancel/{job_id}.

Skip mechanics

Per-platform fallback chain — first path that advances the feed wins:

PathTriggerWhy
1 button[aria-label="Next"].click() Most reliable — user-trusted event
2 synthetic ArrowDown Works even if button is hidden on mobile
3 scrollBy(top: innerHeight) Brute-force fallback against the feed root

Throttled to one skip per 1.5 s. Sensitivity floors: gentle 0.85, balanced 0.78, strict 0.70.
